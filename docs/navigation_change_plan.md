# 画面ナビゲーション変更計画

## 目的
- スマートフォン版でYouTubeのようなボトムナビゲーションを提供し、主要機能への到達性を向上させる。
- Android版はボトムナビゲーションを採用し、Web版はサイドバーで主要導線を維持する。
- 現在のホーム画面から大幅に変更「現在参加できるルーム」をランダムにリスト表示する。

## 対象プラットフォーム
- Android: ボトムナビゲーションを実装。
- Web: サイドバーを実装(ヘッダーとの干渉に注意)。

## 変更概要
1. **ナビゲーションUIの分岐**
   - プラットフォーム判定により、Androidはボトムナビゲーション、Webはサイドバーを表示。
   - 既存のナビゲーション構造を共通化し、UIのみ差し替え可能にする。

2. **ホーム画面の表示内容**
   - 「現在参加できるルーム」をランダム順でリスト表示。
   - ルーム一覧取得ロジックにランダム化オプションを追加。

3. **UI/UXの整合性**
   - ボトムナビゲーションのタブ数・ラベル・アイコンを既存ホーム画面のタイルと一致させる。
   - 画面遷移の状態保持（選択中タブの強調など）を統一。

## 実装方針
- 共通のナビゲーション定義（タブ一覧・ルート）を切り出し、Android/Webで表示コンポーネントを切替。
- ホーム画面のリスト表示は、取得後のランダムソートまたはクエリ側でシャッフルする。
- 未使用警告（unused）の削除は「設計上不要かどうか」を確認した上で対応する。

## タスク一覧（案）
1. ナビゲーション定義の共通化
2. Android向けボトムナビゲーションUIの追加
3. Web向けサイドバーUIの維持・調整
4. ホーム画面のルーム一覧ランダム化
5. 画面遷移/状態保持の動作確認
6. unused警告の棚卸し（必要性を再検討）

## 影響範囲
- ナビゲーションUI
- ホーム画面のルーム一覧取得・表示

## 確認項目
- Androidでボトムナビゲーションが表示され、タブ遷移が正しく動作する。
- Webでサイドバーが表示され、既存動作に影響がない。
- ホーム画面で参加可能ルームがランダムに表示される。
- unused警告の対応方針が設計に合致している。
